System Architecture & Specifications

Multi-source RAG system powered by Qdrant (http://localhost:6333) and Groq LLMs

🤖 Overview & Objectives

QABuddy.AI is a self-hosted, grounded QA assistant engineered for software QA teams. It connects directly to a local Qdrant vector database server at http://localhost:6333 and uses Groq high-speed LLMs. Strict threshold gating guarantees factual precision — responding with "information not found in knowledge base" rather than generating hallucinations.

⚡ Groq LLM (llama-3.3-70b) 📦 Qdrant Server (localhost:6333) 🔢 Dense + Sparse Embeddings ✅ Citation Grounded 🛡️ Hallucination Gated

📦 Qdrant Server Client Integration

The Python backend initializes QdrantClient connecting to local server at http://localhost:6333:

# Python Qdrant Initialization from qdrant_client import QdrantClient client = QdrantClient(url="http://localhost:6333") print(client.get_collections()) # API Health check curl -X GET "http://localhost:6333"

📁 Data Sources & Chunking Strategies

#SourceFormatChunking Strategy
01Selenium POM RepoJavaClass/Method scope parsing. Separate chunks per method with header paths.
02Playwright POM RepoTS/JSBlock scope parsing. Splits by describe / test spec definitions.
03Test Cases DatabaseCSV1 row = 1 chunk. Key-value field serialization preserving column associations.
04JIRA TicketsJSON1 ticket = 1 chunk. Bundles description, status, and comment threads.
05Company GuidelinesPDF / MDHeading-aware recursive splitter. 512-token blocks with 15% overlap.
06Figma Layout SpecsMarkdownSection-based guidelines mapping UI elements and color tokens.
07Meeting TranscriptsText15-line speaker-turn sliding windows with 3-line overlaps.
08Lucid FlowchartsText ExportElement link and connection flow-based text splitting.
09PRDs / RequirementsPDF / MDPage-by-page extraction. Citations carry original PDF page numbers.
10Jenkins Build LogsLogError block extractor capturing stack traces + 5 surrounding context lines.
