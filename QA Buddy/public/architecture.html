📦 Qdrant Client Integration & Payload Schema

Python connection code (`http://localhost:6333`) and Qdrant Point payload structure

Qdrant Code
# Python backend connection to local Qdrant server from qdrant_client import QdrantClient from qdrant_client.http import models # Initialize Qdrant client targeting local server port 6333 client = QdrantClient(url="http://localhost:6333", timeout=3.0) # Recreate collection with dual vector configuration (1024d Dense + Sparse) client.recreate_collection( collection_name="qabuddy_collection", vectors_config={ "dense": models.VectorParams(size=1024, distance=models.Distance.COSINE) }, sparse_vectors_config={ "sparse": models.SparseVectorParams(index=models.SparseIndexParams(on_disk=True)) } ) # Print collection stats collections = client.get_collections() print("Connected Qdrant Collections:", collections)

📁 10 Data Sources & Chunking Strategies

Specific parsing strategies, chunk token sizes, and payload attributes per QA source

Data Pipeline
# Source Name Format Parsing Strategy & Chunking Rule Payload Attributes
01 Selenium POM Repo .java Class & Method block scope parsing. Each `@Test` method or Page Object class becomes a separate chunk. path, line_start, line_end
02 Playwright Spec Repo .ts / .js Block-level AST parsing. Splits by describe() and test() spec definitions. path, line_start, line_end
03 Test Cases Database .csv Row-level serialization. 1 row = 1 chunk preserving column headers (ID, Title, Steps, Expected). tc_id, module, priority
04 JIRA Ticket Exports .json Object mapping. 1 ticket = 1 chunk bundling issue key, description, resolution, and comments. ticket_key, status, assignee
05 Company QA Guidelines .pdf / .md Heading-aware recursive text splitter. 512-token blocks with 15% sliding overlap. path, page, heading
06 Figma Layout Specs .md Section-based guideline parser mapping UI components, color tokens, and layout margins. component_name, spec_id
07 Meeting Transcripts .txt 15-line speaker turn sliding window with 3-line overlap preserving dialogue context. date, speakers
08 Lucid Flowchart Exports .txt / .json Diagram connection flow parsing. Maps element nodes, links, and conditional decisions. diagram_id, flow_name
09 PRDs & Requirements .pdf / .md Page-by-page PDF extraction. Answer citations preserve original PDF page numbers. doc_title, page
10 Jenkins Build Logs .log Error block extractor capturing stack traces + 5 surrounding context lines. build_id, job_name, status
