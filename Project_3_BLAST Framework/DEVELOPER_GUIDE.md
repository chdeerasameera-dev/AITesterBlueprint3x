## 1. Project Overview

BLAST Framework is a **full-stack web application** that:

1. Accepts a **Jira issue ID** (e.g., `SCRUM-5`)
2. Fetches the issue from **Atlassian Jira Cloud** via their REST API
3. Sends the issue details to **GROQ's LLaMA 3** AI model
4. Returns either a **Test Plan** (full document) or **Test Strategy** (focused document)
5. Displays the result in the browser and lets you **download it as Markdown**

### Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React + Vite |
| Styling | Vanilla CSS (3 themes: Light / Dark / Ocean) |
| Backend | Node.js + Express |
| AI | GROQ API (LLaMA 3.3 70B) |
| Data Source | Atlassian Jira Cloud REST API v3 |

## 2. How to Run the Application

### Prerequisites

- **Node.js** ≥ 18.0
- **npm** ≥ 9.0
- A valid **Jira API Token** ([get one here](https://id.atlassian.com/manage-profile/security/api-tokens))
- A valid **GROQ API Key** ([get a free one here](https://console.groq.com/keys))

---

### Step 1: Install Dependencies

Open a terminal in the project folder and run:

```bash
npm install
```

---

### Step 2: Configure Environment Variables

Copy the sample env file and fill in your credentials:

```bash
copy .env.sample .env
```

Edit `.env`:

```env
# GROQ API key (free): https://console.groq.com/keys
GROQ_KEY="gsk_your_full_key_here"

# Jira account email
JIRA_EMAIL="your-email@example.com"

# Jira API Token: https://id.atlassian.com/manage-profile/security/api-tokens
JIRA_API_TOKEN="ATATT3x..."

# Jira base URL
JIRA_URL="https://your-domain.atlassian.net"
```

> **Important:** The GROQ key must be the full key (56 characters starting with `gsk_`).

---

### Step 3: Build the Frontend

```bash
npm run build
```

This compiles the React app into the `dist/` folder that the Express server serves.

---

### Step 4: Start the Server

```bash
node server.js
```

You should see:

```
🚀 Test Plan Buddy server running on http://localhost:3000
📊 API: http://localhost:3000/api/
💻 Frontend: http://localhost:3000/
```

Open your browser at **http://localhost:3000**

---

### Step 5: Generate a Document

1. On the **Generate** page, select either **📋 Test Plan** or **🎯 Test Strategy** using the toggle
2. Enter a Jira issue ID (e.g., `SCRUM-5`)
3. Click **🚀 Generate**
4. View the result in the collapsible sections
5. Click **⬇️ Download Markdown** to save the document

---

### Development Mode (Hot Reload)

To run the frontend with hot-reload during development:

```bash
# Terminal 1: Start the backend
node server.js

# Terminal 2: Start the frontend dev server (runs on port 5173)
npm run dev
```

> When using `npm run dev`, the Vite dev server proxies API calls to the Express server. The proxy is configured in `vite.config.js`.

## 5. How to Switch: Test Plan ↔ Test Strategy

The mode is controlled in **3 places** — UI, backend, and AI prompt. Here's a full map:

### In the UI (no code change needed)
The user selects the mode via the **📋 Test Plan / 🎯 Test Strategy** toggle on the Generate page. This is already implemented.

---

### To change the DEFAULT mode

**File:** `src/components/Generator.jsx` — **Line 11**

```jsx
// Change 'testPlan' to 'testStrategy' to make Strategy the default:
const [mode, setMode] = useState('testPlan');
//                                 ↑ change this
```

---

### To change what sections appear in Test Strategy

**File:** `tools/groqClient.js` — function `buildTestStrategyPrompt()`

Modify the JSON structure in the prompt to add/remove sections. For example, to add `entryCriteria`:

```js
// Add this field to the JSON structure in the prompt:
"entryCriteria": ["<criterion1>", "<criterion2>"],
```

**Then also update the renderer:**

**File:** `tools/testPlan.js` — function `testStrategyToMarkdown()`

```js
// Add the section to the markdown renderer:
if (testPlan.entryCriteria) {
  md += `## Entry Criteria\n\n`;
  md += bulletSection('', testPlan.entryCriteria);
  md += `---\n\n`;
}
```

**And add it to the UI:**

**File:** `src/components/TestPlanView.jsx`

```jsx
// In the "always shown" section (outside the mode === 'testPlan' block):
{testPlan.entryCriteria && (
  <SectionBlock id="entryCriteria">
    {renderArray(testPlan.entryCriteria)}
  </SectionBlock>
)}
```

---

### To rename "Test Strategy" to something else (e.g., "QA Approach")

Change the label in **3 places**:

| File | Where | What to change |
|------|-------|----------------|
| `src/components/Generator.jsx` | `MODES` array (line ~7) | `label: '🎯 Test Strategy'` → `'📊 QA Approach'` |
| `src/components/TestPlanView.jsx` | Badge text (line ~68) | `'🎯 Test Strategy'` → `'📊 QA Approach'` |
| `tools/testPlan.js` | `testStrategyToMarkdown()` heading | `# ${testPlan.title \|\| 'Test Strategy'}` |

**File:** `tools/groqClient.js`

```js
function buildTestCasesPrompt(issue) {
  return `You are a QA engineer. Generate test cases for: ${issue.key}...`;

```

Update the `generate()` function:

```js
const prompt = mode === 'testStrategy'
  ? buildTestStrategyPrompt(issue)
  : mode === 'testCases'
    ? buildTestCasesPrompt(issue)
    : buildTestPlanPrompt(issue);
```

### Step 3 — Add markdown renderer

**File:** `tools/testPlan.js` — add a new renderer function and update `toMarkdown()`:

```js
function toMarkdown(testPlan, mode = 'testPlan') {

  if (mode === 'testStrategy') return testStrategyToMarkdown(testPlan);
  if (mode === 'testCases')    return testCasesToMarkdown(testPlan);  // NEW
  return testPlanToMarkdown(testPlan);
}
```

**File:** `src/components/TestPlanView.jsx` — add new `SectionBlock` renders for the new mode's fields.

## 9. API Reference

### `GET /api/health`

Returns server status and which credentials are configured.

**Response:**
```json
{
  "status": "ok",
  "timestamp": "2026-06-11T...",
  "service": "Test Plan Buddy",
  "configStatus": {
    "hasJiraUrl": true,
    "hasJiraEmail": true,
    "hasJiraToken": true,
    "hasGroqKey": true
  }
}
```

---

### `POST /api/generate`

Generates a test plan or strategy document.

**Request Body:**
```json
{
  "jiraId": "SCRUM-5",
  "mode": "testPlan",
  "config": {
    "jiraUrl":   "https://your-domain.atlassian.net",
    "jiraEmail": "you@example.com",
    "jiraToken": "ATATT...",
    "groqKey":   "gsk_..."
  }
}
```

> `config` is optional if credentials are set in `.env`  
> `mode` is optional — defaults to `"testPlan"`

**Response (success):**
```json
{
  "success": true,
  "testPlan": { "testPlanId": "TP-SCRUM-5", ... },
  "markdown": "# Test Plan ...",
  "filename": "test-plan-SCRUM-5.md"
}
```

**Response (error):**
```json
{
  "success": false,
  "error": "Jira fetch failed (HTTP 401 Unauthorized): ..."
}
```

---

### `POST /api/save`

Saves the markdown to the local `output/` folder.

**Request Body:**
```json
{
  "markdown": "# Test Plan...",
  "filename": "test-plan-SCRUM-5.md"
}
```

> This endpoint is **disabled on serverless** (Vercel). Use browser download instead.

## 10. Troubleshooting

### ❌ `GROQ generation failed (Status 401 Unauthorized): Invalid API Key`

Your `GROQ_KEY` in `.env` is invalid or expired.

**Fix:** Go to [https://console.groq.com/keys](https://console.groq.com/keys), create a new key, and update `.env`.

---

### ❌ `Jira fetch failed (HTTP 401)` or `HTTP 403`

Your Jira credentials are wrong.

**Fix:**
1. Check `JIRA_EMAIL` matches your Atlassian account email
2. Check `JIRA_API_TOKEN` — regenerate at [Atlassian API Tokens](https://id.atlassian.com/manage-profile/security/api-tokens)
3. Check `JIRA_URL` format: must be `https://yourdomain.atlassian.net` (no trailing slash)

---

### ❌ `GROQ generation failed: Could not parse JSON from response`

The AI returned text instead of valid JSON.

**Fix:** This is transient — try again. If it keeps happening, reduce `temperature` in `groqClient.js`:

```js
temperature: 0.4,  // lower = more deterministic
```

---

### ❌ Server shows `hasGroqKey: false` in health check

The `GROQ_KEY` environment variable is not loaded.

**Fix:**
1. Ensure your `.env` file is in the project root
2. Ensure the key is quoted: `GROQ_KEY="gsk_..."`
3. Restart the server after editing `.env`

---

### ❌ Frontend shows blank page after `npm run build`

**Fix:** Ensure you ran `npm run build` successfully before `node server.js`. Check for build errors in the terminal.
